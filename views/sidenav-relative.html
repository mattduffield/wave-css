Key Features of Relative Mode

  • Scoped to Container: The sidenav only affects its parent container, not the entire page.
  • Scoped Overlay: The overlay (when enabled) only covers the container, not the full viewport.
  • Flexible Positioning: Works on both left and right sides.
  • Push Content: Can push content within the container when opening.
  • Multiple Per Page: You can have multiple relative sidenavs on the same page without conflicts.
  • Container Requirements: Parent must have position: relative and overflow: hidden.
  • Backward Compatible: Existing global sidenavs continue to work unchanged.

How to Use Relative Mode

<!-- Parent container must have: position: relative, overflow: hidden -->
<div class="relative overflow-hidden" style="height: 400px;">
  <wc-sidenav relative push push-target=".content" width="250px" overlay>
    <a href="#">Link 1</a>
    <a href="#">Link 2</a>
  </wc-sidenav>

  <div class="content">
    <!-- Your content here -->
  </div>
</div>

Key Attributes:

  • relative - Enables relative positioning mode
  • push - Pushes content when opening
  • push-target - CSS selector for element to push
  • overlay - Shows overlay when open
  • width - Width of the sidenav
  • right-side - Position on right instead of left
