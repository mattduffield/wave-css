Explain Tree Demo

The wc-explain-tree component renders MongoDB explain plan output as a visual stage-by-stage flow diagram.

Indexed Query (IXSCAN + FETCH)

Efficient query using an index. Green summary — docs examined matches docs returned.

Collection Scan (COLLSCAN)

No index used. Red summary — full collection scan examining many more docs than returned.

Sort + Fetch + Index Scan (3 stages)

In-memory sort after index scan. Yellow warning on SORT stage.

$or Query (Branching)

Two parallel IXSCAN branches merged by an OR stage.

Aggregation Pipeline

A $match → $group → $sort → $limit pipeline.
